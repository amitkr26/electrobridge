# Database Architecture

## Overview

BerojgarDegreeWala uses exactly four databases across two providers. This architecture is a locked decision (ADR-001).

## Database Map

| DB | Provider | Role | Tables |
|----|----------|------|--------|
| DB1 | Supabase (Primary) | Core platform + social data | 36 tables |
| DB2 | Supabase (Secondary) | Archive + overflow | 2 tables |
| NEON1 | Neon (Primary) | Analytics + operational logs | 4 tables |
| NEON2 | Neon (Secondary) | Mirror replicas + cache | 2 tables |

## DB1 — Supabase Primary (Core Platform + Social)

Core platform tables and all social/LinkedIn-style features live here because Supabase Auth (`auth.uid()`) is on this project.

## RLS Policy Summary

RLS is enabled on all tables. Key policies:
- **Public read**: opportunities (active), news_articles, company_pages, learning content
- **Owner access**: user_profiles, saved_opportunities, applications, notifications
- **Authenticated write**: feed_posts, messages, connection_requests
- **Service role**: admin operations, scrape writes
